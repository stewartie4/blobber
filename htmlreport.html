Iteration: 1 - 3. Commit Rename
Request Description
In a similar fashion to some databases, a file is not persisted permanently to the blobber until it is comitted. 0Chain keeps it's network fast by storing data off-chain while maintaining the benefits of on-chain storage by comitting the allocation metadata to the blockchain. A commit requires a write marker. Think of this as the latest entry in an audit trail containing the hash of the entire allocation at the time of upload and a link to the previous write market in the form of the hash of the previous allocation state. **This is a major benefit of 0chain - a GDPR compliant, transparent and immutable audit log is a mandatory feature of the platform.** **Requirements to make this request:** Create a new write marker using the reference path: - Set previous allocation root to current allocation root - Traverse through file tree until file path is reached - Add file metadata entry to that directory - recalculate directory hash - recalculate all parent directory hashes - recalculate root directory hash - set allocation root to root directory hash with timestamp - BLS signature sign the request Shard is now successfully uploaded to a blobber. **Now Repeat steps 3, 4 and 5 for each remaining blobber, postman will auto-increment the current blobber and EC shard for you**
